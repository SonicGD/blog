Про интерфейсы

 Posted on October 4, 2017

Реализация интефрейсов в C# тоже сделана не так прямолинейно как в PHP.

Типичный интерфейс:

interface IFoo
{
    void Foo();
}

Обычная реализация выглядит так:

class Foo : IFoo
{
    public void Foo()
    {
      Console.WriteLine("Foo");
    }
}

И мы можем использовать класс вот так:

var foo = new Foo();
foo.Foo(); //работает

Совсем как PHP. Или нет?

Про полиморфизм

 Posted on October 3, 2017

После PHP в C# не сразу очевидно как работает переопределение методов при наследовании. В PHP всё просто:

<?php

class Foo {
    public function test()
    {
        echo 'Foo';
    }
    
    public function proxyTest()
    {
        $this->test();
    }
}

class Bar extends Foo {
    public function test()
    {
        echo 'Bar';
    }
}

$foo = new Foo();
$foo->test(); //Foo
$foo->proxyTest(); //Foo

$bar = new Bar();
$bar->test(); //Bar
$bar->proxyTest(); //Bar

Переопределённый метод заменяет реализацию базового класса целиком и полностью. В C# всё немного веселее.

Авто-свойства

 Posted on March 23, 2017

В C# есть такая удобная штука как авто-свойства. Выглядит так:

public class Foo { 
   public List<string> Bar {get; set; }
}

С их помощью можно удобно ограничивать доступ к свойствам без уродливых геттеров/сеттеров.

Можно так:

public class Foo { 
   public List<string> Bar {get; private set; } //установить можно только внутри класса
}

Или так:

public class Foo { 
   public List<string> Bar {get; } //read-only, выставить значение можно только в конструкторе
}

Также можно использовать как синтаксический сахар для геттеров/сеттеров:

public class Foo
{
    private string _barJson;

    public List<string> Bar
    {
        get { return JsonConvert.DeserializeObject<List<string>>(_barJson); }
        set { _barJson = JsonConvert.SerializeObject(value); }
    }
}

Или даже так:

public class Foo
{
    private string _barJson;

    public List<string> Bar
    {
        get => JsonConvert.DeserializeObject<List<string>>(_barJson);
        set => _barJson = JsonConvert.SerializeObject(value);
    }
}

Очень удобная вещь. Её нехватает в PHP.

Про неймспейсы

 Posted on February 22, 2017

Заметил, что очень нравится то, что в C# можно импортировать namespace целиком.

using My.Super.NameSpace;

В PHP, к сожалению, мы вынуждены импортировать каждый класс по отдельности.

<?php

use my\super\namespace\Foo;
use my\super\namespace\Bar;
use my\super\namespace\FooBar;
...
use my\super\namespace\FooBar83;

В итоге имеем десятки импортов, которые можно было бы заменить одним.
